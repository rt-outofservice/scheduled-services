# Scheduled Services

Standalone Python scripts replacing Claude Code plugin skills. Business logic is algorithmic; AI (claude -p) is invoked only when genuinely needed.

## Directory Layout

```
common/helpers/       - Shared Python modules (logging, telegram, ai)
services/<name>/      - Each service: main script, config.yaml, .bindeps
scripts/              - Utility scripts (migration, etc.)
docs/plans/           - Implementation plans
playbook.main.yaml    - Spot deployment playbook
env.example-main.yml  - Environment variable template
```

## Testing

Every Python script with business logic includes embedded unittest tests, runnable via:

```
uv run python <script>.py --tests
```

Tests use `unittest` and are gated behind `if __name__ == "__main__"` with `--tests` argument parsing.

## Common Helpers

Scripts import shared utilities via sys.path manipulation:

```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parents[2] / "common" / "helpers"))
from log import setup_logging
from telegram import send_telegram
from ai import call_ai
```

## Config

Each service has a `config.yaml` generated by the spot playbook from environment variables. Cron handles all scheduling - scripts execute unconditionally when triggered.

## Error Handling

Never fail silently. All fatal errors are logged AND sent via telegram. Non-fatal errors are logged and appended to the normal telegram message.

## Logging

Daily log files at `~/.logs/scheduled-services/<service_name>/`. 30 most recent files kept per service.

## Commands

- Run tests: `uv run python <script>.py --tests`
- Lint: `uv run ruff check .`
- Format: `uv run ruff format --check .`
- Pre-commit: `pre-commit run --all-files`
